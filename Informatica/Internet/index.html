Компьютерные сети

Локальные сети, Интернет, службы и сетевое хранение данных

Введение в компьютерные сети

Компьютерная сеть — это совокупность устройств, соединенных каналами связи для обмена информацией и совместного использования ресурсов.

Классификация сетей

  • PAN (Personal Area Network)
  • LAN (Local Area Network)
  • CAN (Campus Area Network)
  • MAN (Metropolitan Area Network)
  • WAN (Wide Area Network)

Характеристики LAN

  • Ограниченная территория (дом, офис, здание)
  • Высокая скорость передачи данных
  • Низкая задержка (latency)
  • Полный контроль над инфраструктурой

Сетевая модель TCP/IP

Модель TCP/IP — это практическая основа современного Интернета, состоящая из четырех уровней.

Прикладной
(HTTP, FTP, DNS)
Транспортный
(TCP, UDP)
Сетевой
(IP, ICMP)
Канальный
(Ethernet, Wi-Fi)

Основная идея

Инкапсуляция данных при отправке и декапсуляция при получении. Каждый уровень добавляет свою служебную информацию.

Ключевые устройства LAN

Маршрутизатор (Router)

Функции L3 (сетевой уровень):

  • Межсетевое взаимодействие
  • NAT (Network Address Translation)
  • DHCP-сервер
  • Определение пути передачи пакетов

Коммутатор (Switch)

Функции L2 (канальный уровень):

  • MAC-адресация
  • Создание домена коллизий
  • Высокоскоростная коммутация кадров
  • Сегментация сети

Точка доступа (Access Point)

Реализация стандартов IEEE 802.11 (Wi-Fi):

  • Беспроводное подключение устройств
  • Режимы: AP, Client, Bridge, Repeater
  • Поддержка различных стандартов Wi-Fi

Адресация в сетях: IP и MAC

MAC-адрес

Физический, "жестко" прошитый адрес устройства (L2)

Формат: AA:BB:CC:DD:EE:FF

Уникален для каждого сетевого устройства

IP-адрес

Логический, настраиваемый адрес (L3)

IPv4: 192.168.1.1 (32 бита, 4.3 млрд адресов)

IPv6: 2001:0db8:85a3:0000:0000:8a2e:0370:7334 (128 бит)

Протокол DHCP

Динамическое распределение IP-адресов в сети

Процесс DORA: Discovery, Offer, Request, Acknowledgment

Система доменных имен (DNS)

DNS — это распределенная система, преобразующая доменные имена в IP-адреса

Иерархическая структура

  • Корневые серверы (13 групп по всему миру)
  • TLD-серверы (.com, .ru, .org и т.д.)
  • Доменные серверы (для конкретных доменов)

Типы DNS-записей

  • A - IPv4-адрес
  • AAAA - IPv6-адрес
  • CNAME - каноническое имя (алиас)
  • MX - почтовый сервер
  • NS - сервер имен

Процесс резолвинга

Рекурсивные и итеративные запросы. Пример:

  1. Запрос к локальному DNS-серверу
  2. Запрос к корневому серверу, если ответа нет в кэше
  3. Запрос к TLD-серверу
  4. Запрос к авторитативному серверу домена
  5. Возврат IP-адреса клиенту

Основные службы Интернета

Служба Протоколы Назначение
Всемирная паутина (WWW) HTTP/1.1, HTTP/2, HTTPS (TLS/SSL) Передача веб-страниц, HTML, CSS, JavaScript
Электронная почта SMTP, POP3, IMAP Отправка, получение и хранение электронных писем
Передача файлов FTP, SFTP, SMB/CIFS Обмен файлами между системами
Удаленное управление SSH, RDP, Telnet Удаленный доступ к системам и управление ими

Электронная почта

  • SMTP - отправка писем (порт 25, 587)
  • POP3 - загрузка писем на клиент (порт 110)
  • IMAP - работа с письмами на сервере (порт 143)

Удаленное управление

  • SSH - безопасный удаленный доступ (порт 22)
  • RDP - графический удаленный рабочий стол (порт 3389)
  • Telnet - небезопасный текстовый доступ (порт 23)

Сетевое хранение данных

Локальные решения

NAS (Network Attached Storage)

  • Специализированное устройство, подключаемое к сети
  • Файловый уровень доступа (SMB, NFS, AFP)
  • Простота настройки и использования

SAN (Storage Area Network)

  • Сеть хранения данных
  • Блочный уровень доступа (iSCSI, Fibre Channel)
  • Высокая производительность

Облачные решения

Облачное хранение

  • Яндекс.Диск, Google Drive, Dropbox
  • Синхронизация между устройствами
  • Совместная работа, контроль версий

Модели облачных услуг

IaaS

Инфраструктура как услуга

Пользователь управляет ОС и приложениями

Примеры: AWS EC2, Yandex Cloud Compute

PaaS

Платформа как услуга

Пользователь развертывает приложения

Примеры: Heroku, Yandex Cloud Functions

SaaS

Программное обеспечение как услуга

Пользователь использует готовое приложение

Примеры: Google Workspace, Microsoft 365

Критерии выбора модели

  • IaaS - полный контроль, гибкость, но требуется администрирование
  • PaaS - фокус на разработке, без управления инфраструктурой
  • SaaS - готовое решение, минимальные технические знания

Анализ DNS

nslookup google.com 8.8.8.8

Запрос к конкретному DNS-серверу

Базовые принципы сетевой безопасности

Межсетевые экраны (Firewall)

Фильтрация трафика на основе правил

  • Брандмауэры Windows/Linux
  • Аппаратные межсетевые экраны
  • Правила для входящего/исходящего трафика

Виртуальные частные сети (VPN)

Создание защищенного туннеля через публичную сеть

  • Шифрование передаваемых данных
  • Анонимизация трафика
  • Доступ к корпоративным ресурсам

Важность HTTPS

Защита передаваемых данных между браузером и сервером

  • Шифрование соединения с помощью TLS/SSL
  • Аутентификация сервера
  • Целостность передаваемых данных
  • Никогда не вводите конфиденциальные данные на сайтах без HTTPS!
