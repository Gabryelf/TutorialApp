Знакомство с Django

Django — высокоуровневый Python веб-фреймворк, созданный для быстрой разработки и чистого, прагматичного дизайна. Разработанный в 2003 году, он следует принципу "Batteries Included" - "все необходимое в комплекте".

Быстрая разработка

Создавайте приложения в разы быстрее благодаря готовым компонентам: ORM, админ-панель, аутентификация и многое другое.

Безопасность

Защита от OWASP топ-10 уязвимостей из коробки. CSRF, XSS, SQL Injection и другие угрозы предотвращаются автоматически.

Масштабируемость

Используется такими гигантами как Instagram, Pinterest, Spotify. Отлично справляется с высокими нагрузками.

Django позволяет разработчикам сосредоточиться на бизнес-логике приложения, а не на написании базового функционала, который уже реализован в фреймворке.

Философия Django

Django построен на трех ключевых принципах, которые делают его уникальным и эффективным инструментом для веб-разработки:

DRY Don't Repeat Yourself

Избегайте повторения кода. Каждый фрагмент знаний должен иметь единственное представление в системе. Это уменьшает количество ошибок и упрощает поддержку.

Batteries Included

Все необходимые инструменты уже в комплекте: ORM, админка, аутентификация, кэширование, RSS, sitemaps и многое другое. Не нужно искать и интегрировать сторонние решения для базовых задач.

Explicit is better than implicit

Явное лучше неявного. Код должен быть понятным и предсказуемым. Это упрощает чтение, отладку и поддержку кода другими разработчиками.

Что получаем из коробки:

  • Мощный ORM для работы с базами данных
  • Автоматическая админ-панель
  • Система аутентификации и авторизации
  • Встроенная защита от CSRF, XSS, SQL Injection
  • Система шаблонов с наследованием
  • Поддержка интернационализации
  • Кэширование, RSS, sitemaps

Когда Django может быть избыточным:

  • Микросервисы с минимальной логикой
  • Прототипы, где важна максимальная скорость
  • Приложения с нестандартными требованиями к архитектуре
  • Когда нужен полный контроль над каждым компонентом
  • Очень простые одностраничные приложения

Экосистема Django

Django имеет богатую экосистему библиотек и инструментов, которые расширяют его возможности и делают разработку еще более эффективной:

Django REST Framework

Мощный и гибкий инструмент для создания Web API. Предоставляет browsable API, аутентификацию, throttling, сериализацию и пагинацию.

Django Channels

Расширение для работы с WebSockets, HTTP/2, чат-ботами и другими асинхронными протоколами поверх Django.

Django Debug Toolbar

Панель отладки, показывающая SQL-запросы, кэш, headers, сигналы, шаблоны и многое другое. Незаменимый инструмент для разработки.

Django Celery

Асинхронная очередь задач для фоновой обработки данных, периодических задач, длительных операций и workflow.

Экосистема Django включает более 4000 пакетов на PyPI, что позволяет решить практически любую задачу, связанную с веб-разработкой.

Django vs Flask vs FastAPI

Выбор фреймворка зависит от задач проекта, требований к производительности и предпочтений команды разработчиков. Сравним три популярных решения для Python:

Критерий Django Flask FastAPI
Тип фреймворка Full-stack Microframework API Framework
Скорость разработки Очень высокая Средняя Высокая
Производительность Хорошая Хорошая Очень высокая (асинхронность)
Встроенная админка ✅ Да ❌ Нет ❌ Нет
Встроенный ORM ✅ Да ❌ Нет ❌ Нет
Документация Отличная Хорошая Очень хорошая
Лучший для Сложных приложений, CMS, CRM, социальных сетей Микросервисов, прототипов, простых API Высоконагруженных API, микросервисов

Ключевой вывод

Django идеально подходит для комплексных веб-приложений, где важна скорость разработки и наличие готовых решений. Flask лучше для микро-приложений и прототипов, а FastAPI — для высокопроизводительных API.

Начало работы: Установка и создание проекта

Первым шагом в работе с Django является его установка. Рекомендуется использовать виртуальное окружение для изоляции зависимостей проекта.

Терминал
# 1. Создание виртуального окружения
$ python -m venv venv

# 2. Активация виртуального окружения
$ source venv/bin/activate # Linux/Mac
$ venv\Scripts\activate # Windows

# 3. Установка Django
$ pip install django

# 4. Проверка установки
$ python -m django --version
# Вывод: 4.2.1 (или другая версия)

После успешной установки можно создать новый проект Django с помощью команды django-admin startproject. Эта команда создаст базовую структуру проекта со всеми необходимыми файлами.

Пример кода: Модели Django

Модели данных: Сердце Django приложения

Модели в Django определяют структуру вашей базы данных с помощью Python классов. Django ORM автоматически создает SQL-таблицы на основе этих моделей.

blog/models.py
from django.db import models
from django.contrib.auth.models import User

# Модель категории для статей блога
class Category(models.Model):
    name = models.CharField(max_length=100)
    slug = models.SlugField(unique=True)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.name

# Модель статьи блога
class Post(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    author = models.ForeignKey(User, on_delete=models.CASCADE)
    category = models.ForeignKey(Category, on_delete=models.SET_NULL, null=True)
    is_published = models.BooleanField(default=False)
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        # Сортировка по умолчанию
        ordering = ['-created_at']

    def __str__(self):
        return self.title

После создания моделей нужно создать миграции (python manage.py makemigrations) и применить их к базе данных (python manage.py migrate). Django автоматически создаст соответствующие таблицы в БД.

Пример кода: Представления и шаблоны

Представления (Views) и шаблоны (Templates)

Представления обрабатывают запросы и возвращают ответы. Шаблоны определяют, как данные отображаются пользователю. Вместе они образуют слой представления в архитектуре MVC.

blog/views.py
from django.shortcuts import render, get_object_or_404
from django.views.generic import ListView
from .models import Post

# Class-Based View для списка статей
class PostListView(ListView):
    model = Post
    template_name = 'blog/post_list.html'
    context_object_name = 'posts'
    paginate_by = 10

    def get_queryset(self):
        # Фильтрация только опубликованных статей
        return Post.objects.filter(is_published=True)
blog/templates/blog/post_list.html

Структура Django проекта

Каждый файл в Django проекте имеет свою четкую ответственность. Понимание структуры проекта — ключ к эффективной разработке.

settings.py

Конфигурация проекта: БД, приложения, middleware, шаблоны, статика, безопасность, локализация, логирование.

urls.py

Маршрутизация запросов. Определяет какие URL ведут к каким представлениям. Поддерживает регулярные выражения.

models.py

Модели данных. Определение структуры БД через Python классы (Django ORM). Включая связи, валидацию и бизнес-логику.

views.py

Логика приложения. Обработка запросов, работа с данными, подготовка контекста для шаблонов, API endpoints.

# Основные команды manage.py
$ python manage.py runserver # Запуск сервера разработки
$ python manage.py shell # Django shell (Python + модели)
$ python manage.py createsuperuser # Создание администратора
$ python manage.py collectstatic # Сбор статических файлов
$ python manage.py test # Запуск тестов
$ python manage.py makemigrations # Создание миграций
$ python manage.py migrate # Применение миграций
